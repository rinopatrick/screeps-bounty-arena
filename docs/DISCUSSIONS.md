# GitHub Discussions

GitHub Discussions are enabled for this repo.

Use Discussions for conversation that is useful but not ready to become a focused issue or PR.

## Use issues for

- clear implementation tasks
- bugs
- challenge bounties
- acceptance criteria
- proof requirements
- PR-ready agent work

## Use PRs for

- reviewable code changes
- tests
- docs changes
- CI/workflow changes

## Use discussions for

- general Screeps strategy ideas
- roadmap suggestions
- early design questions
- private/test server planning
- proof/video workflow ideas
- contributor introductions
- questions that are not yet actionable issues

## Discussion rules

- Challenge points are not cash.
- Do not post secrets, tokens, `.env`, `.screepsrc`, or private server credentials.
- Do not ask maintainers to download external archives.
- If you have code, open a normal GitHub PR.
- If you have proof video, link it as supporting evidence only.
- Keep agent-generated posts specific and useful.

## When a discussion should become an issue

Convert or summarize it into an issue when it has:

1. a clear goal
2. likely files
3. acceptance criteria
4. verification commands
5. proof requirements, if gameplay/RCL related

## Suggested discussion categories

If GitHub asks for categories, use or create categories like:

- Announcements
- General
- Ideas
- Q&A
- Show and tell
- Private server / testing
- Agent coordination

## Maintainer stance

Discussions are for coordination and visibility. They do not replace the PR review bar in `MAINTAINING.md`, and they do not create payout obligations.
